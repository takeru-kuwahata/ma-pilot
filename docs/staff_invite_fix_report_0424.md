# スタッフ招待メール 修正対応のご報告（2026-4-24）

## ⑦　スタッフ招待メールが届かない問題を修正しました

スタッフ管理画面からスタッフを招待しても、招待されたスタッフにメールが届かない問題を修正しました。

**原因**
以前の実装は「管理者がパスワードを設定してアカウントを即時作成する」方式であり、メール送信は行われていませんでした。画面上には「招待リンクが送信されます」と表示されていましたが、実際にはメールが送信されていませんでした。

**修正内容**
Supabaseの招待メール機能を使う方式に変更しました。スタッフ追加フォームから「初期パスワード」入力欄を削除し、ボタンを「招待メールを送信」に変更しました。

**修正後の動作**
1. 管理者がスタッフのメールアドレスと権限を入力して「招待メールを送信」をクリック
2. 招待されたスタッフのメールアドレスに「You have been invited」メールが届く
3. スタッフがメール内の「Accept the invite」リンクをクリックして自分でパスワードを設定
4. 設定したパスワードでログイン

今後は管理者がパスワードを設定・共有する必要はありません。

本番環境でtakerutic@gmail.com宛に招待メールが届くことを確認済みです（2026-4-24）。
