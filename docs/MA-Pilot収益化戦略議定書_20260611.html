歯科医院は「飽和市場の中で静かに崩壊が進む業界」である。コンビニより多い6万院が過当競争にさらされ、廃業は急加速。後継者不在率90%超という構造的問題は、今後10年で数千院単位の廃業・M&Aが発生することを意味する。同時に、生き残りを目指す院長ほど経営データへの渇望が高い。この「危機感とデータ需要のギャップ」がMA-Pilotの存在意義である。

業界の追い風4点
  • 予防歯科の浸透：歯科検診受診率が初めて6割超（2025年）。リピート患者が増え、データ管理ニーズが高まる
  • 自費診療シフト：全国平均21.4%の自費率を40%以上に高めることが院長最大の関心事
  • 診療報酬改定（2026年6月）：「電子的診療情報連携体制整備加算」新設。クラウドSaaS需要が政策的に後押しされる
  • IT導入補助金：最大450万円・1/2補助。SaaS導入の初期コスト障壁を大幅に下げられる

差別化ポジション

現行競合の空白地帯は「経営判断の支援」である。レセコンは診療業務、SaaSは患者管理、コンサルは人的支援と、いずれも「院長が月次で経営数値を見て意思決定する」機能を持たない。MA-Pilotはこの空白を埋める。特に診療圏分析・経営シミュレーション・レポート自動生成の組み合わせは競合に存在せず、強固な差別化ポイントとなる。

院長の購買行動と「売れる理由・売れない理由」

院長の意思決定の構造（調査で判明）
  • 信頼関係が最優先：見知らぬ営業よりも「知人の紹介」「既存業者の推薦」が圧倒的に有効
  • 決定速度は速い：個人経営のため稟議不要。ただし信頼構築フェーズに時間がかかる
  • 情報源：デンタルショー・展示会・勉強会（同時開催セミナー200件超）が主要接触点
  • 費用感：月額3万円以下は「試してみる」域。10万円超は「成果の確信」が必要
  • 最大関心事：自費診療率向上・患者リピート率向上・スタッフ採用
「売れない」典型的失敗パターン
  • 機能説明中心の営業 → 院長は「で、何が解決するのか」しか聞いていない
  • メール・DM単独 → 歯科業界は閉鎖的。初回接触はリアル or 紹介が必須
  • 成果事例なしで高額提案 → 信頼資産のない段階での高額化は致命的
  • サポート体制の曖昧さ → 「使い方がわからない」でそのまま解約

# 仮説 推奨度 初期貢献度 スケーラビリティ 判定
A パートナー企業（歯科商社）経由で50〜100医院獲得 2/5 中（10〜30医院が現実的上限） 低（商社のモチベーション維持困難） 条件付き
B 会長ネットワーク直販で10〜20医院獲得→成功事例化→横展開 4/5 高（3〜6ヶ月で現実的） 中（20院以降は別チャネル必要） 採用推奨

仮説A（パートナー商社展開）— なぜ脱落したか

歯科商社の営業はハードウェア販売が主業であり、SaaSの複雑な説明・サポートを再現することは構造的に困難。代理店SaaSの失敗調査（10社中成功2社）でも「パートナーが複数いるが成果ゼロ」「教育コスト過大」「インセンティブ不足で優先度下落」が主因。展開初期にパートナー頼みにすると、製品の不十分な説明による顧客不満→ブランド毀損リスクが高い。パートナーチャネルは成功事例を作った後の「スケール手段」として位置づけるべき。

仮説C（コンサル付帯高単価）— なぜMVP限定か

初期ARR1000万円達成手段としては有効。しかし月額5〜10万円×月1セッション体制は1人運営の場合ARR上限4500万円（45院）で頭打ち。スタッフ増員で利益率が急落し、SaaSとしての拡張性を失う。初期の10〜20院の試験運用段階では「経営診断セッション付き」の高単価モデルでROIを実証し、30院以降はAI化・プロダクト化（オプション化）に切り替える二段階設計が正解。

確定戦略ロードマップ — Phase 1〜3

推奨戦略：「信頼資産先行型・3フェーズ展開」
会長ネットワークで最初の10〜20医院に深く入り込み、定量的な成功事例を作る（Phase 1）。その事例を武器にパートナー・勉強会チャネルで横展開（Phase 2）。50院超でプロダクト改良・価格体系見直し・補助金活用で加速（Phase 3）。
焦って広げるのではなく、最初の20院を「証明の場」として徹底的に活用することが最速ルート。

Phase 1
0〜6ヶ月

信頼事例の創出 — 目標：10〜20医院・MRR30〜60万円

  • 会長ネットワーク直販：関係のある歯科医院に絞り、特別価格（月額1〜2万円）または3ヶ月無料で導入させる。見返りは「成功事例の証言・推薦」のみ
  • 伴走支援を密に：月1回のオンライン確認セッションを無償提供。「使いこなせない」による早期離脱を防ぐ
  • ROIの記録：導入前後の自費率・患者数・スタッフ稼働効率を数値化。「この医院では3ヶ月で自費率が◯%上がった」という事例を最低3件作る
  • PDFレポートPDFバグ修正：本番でPDFダウンロード失敗の報告あり。Phase 1開始前に必ず解消する
  • IT導入補助金の活用：補助金対象事業者への申請サポートを初期サービスに含める（選ばれる理由になる）
Phase 2
6〜18ヶ月

事例を武器にした横展開 — 目標：50〜100医院・MRR150〜300万円

  • 勉強会・展示会への登壇：デンタルショー・歯科経営セミナーで成功事例を発表。1回の登壇で10〜30名の院長に接触可能
  • パートナー商社への提案（限定的）：Phase 1の成功事例が揃った段階でパートナーを2〜3社に絞り導入。広く薄く展開ではなく、深く支援できるパートナーのみ
  • Lstep Webhook活用：既に実装済みのLINE連携を活かし、見込み医院へのナーチャリング自動化
  • 価格の正規化：月額2.5〜3.5万円（SaaS標準帯）に設定。IT補助金活用で実質負担を1.25〜1.75万円に見せる
  • Stripe決済の本稼働：Phase 2前に環境変数設定を確認・解消（現在待機中）
Phase 3
18〜36ヶ月

プロダクト強化とスケール — 目標：200〜500医院・ARR1億円

  • AI経営診断機能：月1セッションをAIによる自動診断レポートに置き換え。人的コストなしで「コンサル代替価値」を維持
  • レセコン連携：主要レセコンとのAPI連携で自動データ取込を実現。手動入力の手間がチャーンの最大原因になる
  • 価格二層化：基本プラン（月額1.5〜2万円）＋プレミアムプラン（月額4〜5万円・AI診断+診療圏分析フル機能）
  • M&A・事業承継支援機能：廃業急増の市場で「経営評価・承継シミュレーション」を追加すると高付加価値

VII

致命的リスクと撤退条件

致命的リスク 1 — プロダクト品質

現在、本番でPDFダウンロード失敗の未解決バグが存在する。レポートPDFはMA-Pilotの核心機能であり、院長への初回デモ・提案時に動かなければ一発で信頼を失う。Phase 1開始前に必ず解消すること。会長ネットワークへの展開は「プロダクトが動く」ことが大前提。

致命的リスク 2 — 最初の成功事例が出ない

仮説Bの根幹は「成功事例の作成」である。最初の10〜20院でROIが出なければ、Phase 2への横展開は不可能になる。導入後3ヶ月は伴走を密にし、「自費率◯%向上」「無駄コスト◯万円削減」という具体数値を必ず記録・文書化する。数値がなければ営業トークが「自称」になり横展開は動かない。

注意リスク 3 — レセコン大手の参入

WiseStaff・DENTISなどレセコン系が経営分析機能を強化した場合、データ統合の優位で競合不可になる。対策：レセコンとの「競合」ではなく「連携」ポジションを取る。CSV取込・API連携を先行実装し、「レセコンと組み合わせて使うもの」として認知させる。

撤退条件の明示

以下の条件が重なった場合は戦略の全面見直しを行う： ①Phase 1（6ヶ月）終了時に5院以下しか導入できていない、 ②初期10院のうち3院以上が6ヶ月以内に解約、 ③パートナー商社3社にアプローチして全社が断った。 いずれかが発生した場合は価格・機能・ターゲット設定の根本を見直すことを推奨する。
