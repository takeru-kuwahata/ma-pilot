# MA-Pilot デザインガイドライン

## 1. デザインの基本原則

### 1.1 コンセプト
- **シンプル**: 機能を主張せず、使いやすさを最優先
- **ベーシック**: 流行に左右されない、企業向けシステムとしての品格
- **控えめ**: デザインが前に出すぎず、データと機能が主役

### 1.2 対象ユーザー
- 歯科医院の院長・スタッフ（経営データを扱う専門職）
- システム管理者（Medical Advance社）

### 1.3 デザイン指針
- **ログインページ等の公開ページ**: デザインを主張しない、最もシンプルな形
- **管理画面**: データの可読性を最優先、視覚的ノイズを排除
- **グラフ・レポート**: 情報の理解を助けるビジュアライゼーション

## 2. カラーパレット

### 2.1 コーポレートカラー（プライマリ）

```yaml
プライマリカラー（オレンジ）:
  メイン: #FF6B35
  ホバー: #E55A28
  ライト: #FFA07A

使用箇所:
  - 主要アクションボタン（ログイン、保存、送信など）
  - アクティブなタブ・下線
  - リンクテキスト
  - 入力フォーカス時のボーダー
  - 重要な通知・アラート

使用禁止:
  - 大きな面積の背景色（カード全体、ヘッダー背景など）
  - 複数のボタンに同時使用（主要アクション1つのみ）
```

### 2.2 ニュートラルカラー

```yaml
背景色:
  - 白: #FFFFFF（カード、フォーム背景）

  - ミディアムグレー: #E0E0E0（ボーダー、区切り線）

テキスト:
  - 主要テキスト: #424242
  - 副次テキスト: #616161
  - 無効状態: #9E9E9E
  - プレースホルダー: #BDBDBD
```

### 2.3 セカンダリカラー

```yaml
情報表示:
  - ブルー: #1976D2（情報アイコン、セカンダリボタン）
  - グリーン: #388E3C（成功メッセージ）
  - イエロー: #F57C00（警告）
```

## 8. ページタイプ別デザイン

### 8.1 公開ページ（ログイン、パスワードリセット等）

```yaml
特徴:
  - 最もシンプルなデザイン
  - 中央揃えの単一カード
  - 背景はライトグレー（#F5F5F5）
  - カード幅: max-width 400px
  - ロゴ + サービス名のみヘッダー

禁止事項:
  - 過剰な装飾
  - 大きな背景画像
  - 複雑なレイアウト
  - 派手なアニメーション
```

### 8.2 ダッシュボード

```yaml
特徴:
  - データ視認性を最優先
  - カードベースのグリッドレイアウト
  - グラフは色分けを最小限に
  - KPI数値は大きく明確に

配色:
  - グラフ: 最大3色まで
  - 背景: 白とライトグレーのみ
  - アクセント: オレンジは重要な数値のみ
```

### 8.3 フォームページ

```yaml
特徴:
  - シンプルな縦並びレイアウト
  - セクション区切りは控えめ
  - エラー表示は赤（#D32F2F）
  - 必須項目は「*」マーク

ボタン配置:
  - 主要アクション（保存）: 右下
  - セカンダリ（キャンセル）: その左
  - 破壊的アクション（削除）: 左端
```

## 9. アニメーション・トランジション

### 9.1 基本方針

```yaml
原則:
  - 控えめで自然な動き
  - ユーザーの操作を妨げない
  - 不必要なアニメーションは避ける
```

### 9.2 トランジション時間

```yaml
高速: 150ms（ホバー、フォーカス）
標準: 250ms（モーダル表示、ドロワー開閉）
低速: 400ms（ページ遷移）

イージング:
  - 標準: ease-in-out
  - 表示: ease-out
  - 非表示: ease-in
```

## 10. アクセシビリティ

### 10.1 色のコントラスト

```yaml
WCAG AA基準準拠:
  - テキスト（標準）: 4.5:1以上
  - テキスト（大）: 3:1以上
  - UI要素: 3:1以上
```

### 10.2 フォーカス表示

```yaml
必須:
  - キーボード操作でフォーカス可視化
  - outline: 2px solid #FF6B35
  - outline-offset: 2px
```

### 10.3 代替テキスト

```yaml
必須:
  - すべての画像・アイコンにaria-label
  - グラフ・チャートに代替テキスト
  - エラーメッセージはスクリーンリーダー対応
```

## 12. チェックリスト

### 12.1 デザインレビュー時の確認項目

```yaml
配色:
  - [ ] コーポレートカラーは適切に使用されているか？
  - [ ] 大きな面積にオレンジを使っていないか？
  - [ ] コントラスト比はWCAG AA基準を満たしているか？

レイアウト:
  - [ ] 余白は8pxグリッドに沿っているか？
  - [ ] ページタイプに適したレイアウトか？
  - [ ] レスポンシブ対応がされているか？

タイポグラフィ:
  - [ ] フォントサイズは規定通りか？
  - [ ] 行間は適切か？
  - [ ] フォントウェイトは過剰でないか？

コンポーネント:
  - [ ] MUI標準コンポーネントを使用しているか？
  - [ ] カスタマイズは必要最小限か？
  - [ ] 一貫性が保たれているか？

シンプルさ:
  - [ ] 不要な装飾を排除しているか？
  - [ ] デザインが機能より前に出ていないか？
  - [ ] ユーザーは迷わず操作できるか？
```
