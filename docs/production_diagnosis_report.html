🚨 必須対応事項（Critical / High）

  • 🔴 [セキュリティ] CSP設定の不備 — unsafe-inline / unsafe-eval 許可
    Content-Security-Policyに'unsafe-inline'と'unsafe-eval'が含まれており、XSS攻撃のリスクを大幅に増加させています。nonceまたはhashベースのCSPへの移行が必要です。
    backend/main.py:77-84
  • 🔴 [セキュリティ] JWTトークンのlocalStorage保存
    アクセストークンをlocalStorageに保存しており、XSS攻撃によるトークン窃取のリスクがあります。HttpOnly Cookieまたはメモリ内保存への変更を推奨します。
    frontend/src/services/api/authService.ts:24
  • 🔴 [セキュリティ] CORS allow_headers='*'（全ヘッダー許可）
    全ヘッダーを許可するワイルドカード設定になっています。必要なヘッダーのみをホワイトリスト化してください。
    backend/main.py:117
  • 🔴 [信頼性] トランザクション管理が完全に不在
    ユーザー登録（Auth+clinics+user_metadata）、印刷物注文作成（print_orders+print_order_items）等の複数テーブル操作にトランザクション管理がなく、部分失敗時にデータ不整合が発生します。
    backend/src/services/auth_service.py:86-142, print_order_service.py:151-287
  • 🔴 [信頼性] グローバルエラーハンドラーが不在
    FastAPIアプリにグローバルなException/HTTPException/ValidationErrorハンドラーがなく、予期しないエラー時にスタックトレースが外部に露出する可能性があります。
    backend/main.py（ハンドラー追加が必要）
  • 🔴 [パフォーマンス] N+1問題（注文一覧API）
    注文一覧取得時に各注文ごとにループ内でDBクエリを実行するN+1問題が存在します。注文数に比例してクエリ数が増加します（100件=101クエリ）。order_idのIN句による一括取得に変更してください。
    backend/src/services/print_order_service.py:310-320
  • 🔴 [コード品質] テストの73%がスキップ状態
    バックエンドテストの50件がskipされており、実質的なカバレッジが20-30%程度と推定されます。test_auth.py、test_dashboard.py、test_clinics.py等の実装が必要です。
    backend/tests/（多数のskipデコレータ）
